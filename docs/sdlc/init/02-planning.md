# Планирование — System Design Trainer

## Scope

### В скоупе (Фаза 1 — MVP)
- Инициализация проекта Next.js + TypeScript
- Типы данных для вопросов
- Quiz-вопросы по 2-3 темам из каждого блока
- Базовые компоненты: карточка вопроса, варианты ответа, результат
- Стейт-машина квиза (useQuizSession)
- Страница выбора темы → страница квиза → результат

### В скоупе (Фаза 2)
- Открытые вопросы + OpenAI оценка + fallback
- Все темы и вопросы (полное наполнение)
- Таймер, прогресс (localStorage)
- Dashboard
- Фильтрация по сложности

### Вне скоупа (backlog)
- Авторизация и синхронизация прогресса
- Тёмная тема
- Мобильное приложение
- Многоязычность

## Критерии приёмки

| ID | Критерий | Фаза |
|----|----------|------|
| AC-1 | Пользователь может выбрать блок и тему | 1 |
| AC-2 | Quiz-вопрос отображается с вариантами | 1 |
| AC-3 | После ответа показывается результат и объяснение | 1 |
| AC-4 | Переход к следующему вопросу | 1 |
| AC-5 | Открытый вопрос с текстовым полем | 2 |
| AC-6 | ChatGPT оценивает ответ | 2 |
| AC-7 | Fallback при отсутствии API-ключа | 2 |
| AC-8 | Прогресс сохраняется между сессиями | 2 |
| AC-9 | Таймер обратного отсчёта | 2 |
| AC-10 | Dashboard с общей статистикой | 2 |

## Риски

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| OpenAI API недоступен | Средняя | Graceful fallback, не блокер |
| Вопросы устаревают | Низкая | Вопросы в коде, легко обновить |
| Сложность наполнения контентом | Высокая | Генерация контента при разработке |
