# Требования — System Design Trainer

## Функциональные требования

### FR-1: Два блока вопросов
- **Системный Анализ (SA)**: нотации моделирования, архитектура и интеграции, базы данных, безопасность
- **System Design (SD)**: масштабируемость, балансировка нагрузки, кэширование, БД, очереди сообщений, CDN, API Design, микросервисы, CAP-теорема

### FR-2: Типы вопросов
- **Quiz**: вопрос + варианты ответа (один правильный)
- **Открытый вопрос**: пользователь пишет текстовый ответ, оценивается ChatGPT

### FR-3: Уровни сложности
- Junior, Middle, Senior

### FR-4: Корректные ответы
- Каждый вопрос содержит эталонный ответ и развёрнутое объяснение
- После ответа пользователь видит правильный ответ и пояснение

### FR-5: AI-оценка открытых ответов
- Используется OpenAI API (ChatGPT) для развёрнутой оценки
- Если API-ключ отсутствует или нет соединения — заглушка с предложением сравнить ответ с эталонным

### FR-6: Прогресс
- Сохраняется в localStorage (работает без логина)
- Опционально: авторизация для синхронизации (фаза 2+)

### FR-7: Таймер
- Режим таймера для имитации давления собеседования

### FR-8: Dashboard
- Общая статистика: пройдено вопросов, % правильных, прогресс по темам

## Нефункциональные требования

### NFR-1: Язык интерфейса
- Русский

### NFR-2: Платформа
- Web-приложение, Next.js App Router

### NFR-3: Офлайн-устойчивость
- Приложение работает без интернета (кроме AI-оценки)

### NFR-4: Производительность
- Страницы загружаются < 2 сек
- Вопросы статичны, не требуют запросов к БД

## Целевая аудитория
- Системные аналитики уровня Senior/Lead, готовящиеся к собеседованиям
