## Acceptance Criteria

| ID | Критерий | Milestone |
|----|----------|-----------|
| AC-1 | Тип `Block` включает `'ai'` | M1 |
| AC-2 | На главной странице отображается карточка AI с количеством вопросов и списком тем | M1 |
| AC-3 | В настройках тренировки доступна кнопка «AI» для выбора блока | M1 |
| AC-4 | Каждая тема содержит 8–10 вопросов (mix quiz + open) | M2–M6 |
| AC-5 | Каждая тема покрывает все 3 уровня сложности | M2–M6 |
| AC-6 | ID вопросов соответствуют формату `ai-{topic}-{NNN}` | M2–M6 |
| AC-7 | Фильтрация по теме и сложности работает для блока AI | M1 |
| AC-8 | Прогресс по AI сохраняется в localStorage | M1 |
| AC-9 | Все существующие тесты проходят | M1 |
| AC-10 | `npm run build` завершается без ошибок | M6 |

## Зависимости

```
M1 (инфраструктура) ← M2, M3, M4, M5, M6 (контент)
```

Файлы с контентом зависят от M1 (тип `Block` и импорты в `questions.ts`), но между собой независимы. Контент можно создавать параллельно после M1.

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|------------|---------|-----------|
| Большой объём контента (~120 вопросов) | Высокая | Средняя | Параллельная генерация по темам, строгий шаблон |
| Неточности в техническом контенте | Средняя | Высокая | Ревью каждой темы, ссылка на актуальные источники |
| Grid layout ломается при 3 карточках | Низкая | Низкая | Адаптивный grid: 1→2→3 колонки |
| Устаревание контента (AI меняется быстро) | Средняя | Средняя | Вопросы в коде, легко обновить |
